# US-005 — Add operator-facing next-action summaries from demand context

**Date:** 2026-03-29  
**Story ID:** US-005  
**Story Title:** Add operator-facing next-action summaries from demand context

---

## Summary

Extended the operator-facing `/admin/strategy` surface to include a concrete,
per-municipality **next-action recommendation** that combines the verified
demand signal (US-001/US-002) with the outreach targeting score (US-003).
Each row in the strategy view now tells the operator exactly what to do next
for that municipality — replacing generic analytics badges with actionable
language tied to real resident behavior and the municipality's energy-
transition profile.

---

## Problem

The `/admin/strategy` endpoint exposed demand levels and scores but left
operators to interpret those numbers themselves.  There was no mechanism that
translated "demand_level = high, outreach_score = 72" into a concrete
recommendation such as "Contact municipality administrator to initiate LEG
formation — strong resident demand is confirmed."

---

## Changes Made

### `insights_engine.py`
- **Added `compute_next_action_summary(demand_level, demand_score, outreach_score, verified_buildings, communities_in_formation) → str`**  
  Translates demand level + outreach targeting score into a single, concrete
  operator recommendation.  Logic:

  | Demand level | Condition | Recommendation |
  |---|---|---|
  | `high` | Formation already active | Accelerate onboarding, coordinate with utility |
  | `high` | No formation yet | Contact municipality administrator immediately |
  | `medium` | outreach_score ≥ 50 | Schedule municipality meeting now |
  | `medium` | outreach_score < 50 | Send awareness content / invite to pilot workshop |
  | `low` | outreach_score ≥ 60 | Launch prospecting campaign despite low signal |
  | `low` | outreach_score < 60 | Monitor registrations; send periodic updates |
  | `none` | outreach_score ≥ 60 | Run initial awareness campaign (strong profile) |
  | `none` | outreach_score < 60 | Maintain watch; revisit when registrations arrive |

### `app.py`
- **Updated `/admin/strategy` route:**  
  - Now also calls `db.get_all_municipality_profiles()` and
    `rank_municipalities_for_outreach()` to obtain per-municipality targeting
    scores.
  - Enriches each signal dict with `outreach_score` and `next_action` fields
    before returning JSON or rendering HTML.
  - Gracefully handles missing profile data (DB error → empty profiles list;
    `outreach_score` defaults to 0.0).

### `templates/admin/strategy.html`
- Added **"Outreach Score"** column (purple highlight, 0–100) between the
  demand score and verified buildings columns.
- Added **"Next Action"** column as the rightmost column — renders the
  recommendation text per municipality row.
- Updated empty-state `colspan` from 9 to 11.

### `tests/test_next_action_summary.py` *(new file)*
- 9 unit tests for `compute_next_action_summary()` — one per branch plus
  a parametric smoke test across all levels.
- 5 integration tests for the enriched `/admin/strategy` route — verifying
  JSON contract (`next_action`, `outreach_score` fields), text content for
  high-demand entries, HTML column rendering, and DB-error fallback.
